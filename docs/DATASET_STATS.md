# RakshakAI v2 — Dataset Statistics

> **Status:** Phase 2 (data engineering only — no GPU).
> **Last updated:** 2026-06-07

This document is the source of truth for *what is in the v2 training set right now*, how balanced it is, and *what the production target looks like* once the real Tier A/B sources are downloaded.

---

## 1. Current state (this PR)

The v2 corpus is currently a **balanced seed** of 250 samples drawn from:

- `v1-augmented` — 172 samples generated by text-augmenting 41 unique v1 templates
- `tier-b-seed` — 78 samples from 21 hand-curated multi-language examples

The full production corpus (downloaded from BigVul, Devign, PrimeVul, SecurityEval, Juliet, CWE-699, CVEfixes, etc.) is documented in [`docs/DATASET_SOURCES.md`](DATASET_SOURCES.md) and reuses the same pipeline; the per-source `*.jsonl` files in `v2/inputs/datasets/raw/` are the integration point.

## 6. Reproducing these numbers

```bash
# 1. Re-convert v1 CSV to v2 JSONL
PYTHONPATH=. python3 v2/dataset/adapters/from_v1_csv.py

# 2. Generate Tier B multi-language seed
PYTHONPATH=. python3 v2/dataset/adapters/seed_tier_b.py

# 3. Text-augment the v1 templates
PYTHONPATH=. python3 v2/dataset/adapters/augment.py --variants 30

# 4. Clean (dedup, validate, filter)
PYTHONPATH=. python3 v2/dataset/clean.py --near-dup-threshold 0.92

# 5. Balance + split
PYTHONPATH=. python3 v2/dataset/balance.py --max-per-class 50

# 6. Inspect the balance report
cat v2/inputs/datasets/balanced/_balance_report.json
```

All steps are deterministic and idempotent.
